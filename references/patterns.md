## 姓名/组织名提示（仅 `info`，需要人工复核）

- `^\s*(?:作者|Author|By|by|Maintainer)\s*[:：]\s*...` — 真实姓名
- 常见学校名（武大/清华/...）+ 常见公司名（Microsoft/Google/...）— 仅在 README/注释里出现时算

**为什么不做强匹配**：中英文姓名形式太多，机构名只能列举；过度匹配会把项目里"Google 地图 SDK"等合法引用也标红。让 agent 看完人工二次确认。

## 跳过的目录

```
.git, node_modules, __pycache__, .venv, venv, env,
dist, build, target, .next, .nuxt, .output,
vendor, Pods, .gradle, .idea, .vscode,
.mypy_cache, .pytest_cache, .ruff_cache, .tox,
Library, DerivedData, bin, obj
```

**注意**：`.git` 在文件扫描时被跳过，但 `--scan-history` 模式会从 `.git` 读 `git log -p` 单独扫描历史。两者互补。

## 跳过的内容扫描

- 大于 10 MB 的文件：只报 large_file，不扫内容
- 二进制文件（无扩展名匹配 SCANNABLE_EXTS 的）

## 历史扫描（`--scan-history`）

走 `git log --all -p -n <max-commits>`，对每条 commit 的 `+` 添加行跑同一套 pattern。

输出格式与文件扫描一致，但：

- `path` 字段是 `commit:<sha12>:<file>:<line>`（如 `commit:a8b224f678de:config.py:1`）
- `snippet` 前缀是 `[author <email> <subject>]`
- `suggestion` 改成"考虑 amend/rebase 此 commit，或用 `git filter-repo` 重写历史"

默认 `--max-commits=200`，可调。

## 怎么扩展

想加新模式时直接改 `scripts/scan.py` 顶部的列表，保持类别和严重度一致即可。**改完跑一遍当前项目 + 合成测试做回归**。
